第一步：DaVinci 批量生成 Proxy
- 导入你的 Cinema DNG 或者 RAW 文件
- 进到 Deliver 页面
- 设置输出格式选 ProRes Proxy 或 H.264 low bitrate（看你机器性能）
- 勾选 Batch Export All Clips
- 输出路径记得指定一个专门放 proxy 的文件夹
- 点击 Start Render，一杯咖啡的时间就搞定🔥

第二步：DaVinci 直接导出 FCP XML
- 在 Deliver 页面最下面有个 Generate Final Cut Pro XML 的选项，勾上✔️
- 它会自动把你的时间线结构、转场、调色统统打包成 XML 文件
- 把这个 XML 导入 FCP 后，它默认会先加载 proxy 视频，剪起来丝滑到飞起😎

第三步：Proxy 回套原片的 trick
- 最关键的是——你在 DaVinci 输出 proxy 的时候一定要保留原始时间码（Timecode）和文件名一致✅
- FCP 里用“Reconnect Media”功能，批量搜索原片文件夹，FCP 会自动根据时间码或文件名匹配原片
- 如果你用的是双硬盘结构（proxy盘 + 原片盘），记得都连上再做回套，不然容易 missing file😭

我一般还会加个小技巧：
- 在 FCP 工程里设置一个快捷键，比如 `F1` 切换 proxy / original（通过切换 Project Properties 中的 Optimized/Proxy Media 路径）
- 这样你在剪辑时用 proxy，导出前一键切回原片，效率起飞🚀

### 🎬 Insta360 + Proxy Workflow 整合方案：

#### 1. Insta360 Studio 批量降采样导出Proxy
- 导入你的360°素材后，进到 Batch Export 页面
- 设置分辨率选 1080p（或者720p也行，看你proxy用途）
- 编码选 H.264 或 ProRes Proxy（Mac用户优先）
- 输出路径单独建个 `/proxy` 文件夹
- 勾上 Export Reframe Data Only（这样DaVinci可以继承镜头运镜轨迹）✅
- 一键全部render，省时又高效💪

#### 2. DaVinci Autopilot + Proxy联动
- 把proxy视频导入DaVinci → 进到 Edit 页面
- 右键点击素材 → Stabilize & Reframe（启用Autopilot Reframe）
- 调整好构图、运动轨迹后，别急着导出！先勾选 Use Proxy Mode 👌
- 这样它会记住你reframe的参数，但不会浪费资源跑高清渲染

#### 3. XML回套Final Cut Pro
- 在Deliver页面，记得勾选 Generate Final Cut Pro XML
- 同时保留原始文件名和时间码（这点超级关键！）
- 导出XML后丢进FCP，再用之前说的Reconnect Media技巧一键切换回原片
- 最终输出的时候就直接跑高分辨率版本，效率起飞🚀
